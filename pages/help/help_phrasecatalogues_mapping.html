Help Pages of the Phrasetool by Rimpido

A Phrasecatalogue is a collection of Phrases, which consist of Phraseheads (its base data) and Phrasetexts (its translations), gorup by PHrasegroups. The Phrasecatalogue has some base data which can be edited. Furthermore, the Phrasecatalogues can be mapped against each other, i.e., the Phrases will be compared and if one Phrase from the origin catalogue is sufficiently congruent with a Phrase of a target catalogue a reference wil be written. Such Mapping can be verified via Phrasencatalogue- Comparison.

The help of the Phrasecatalogue contains of the following topics:

  • Overview of Catalogues
  • Editing of Catalogue
  • Mapping of Catalogues
  • Comparison of Catalogues

    • Overview of Catalogues

    The index page gives an overview over all used catalogues (of your company). Only catalogues visible on this page may be used for further functions. These functions are "View", "Edit", "Mapping", and "Delete". Not every user is allowed to use all function. These functions are explained below. Please note: Adding a phrasecatalogue is nearly similar to editing existing ones, so please look for information there.

    Editing of Catalogues (And Adding)

    For Phrasecatalogues only some six variables must be set.

    Katalog Pin: This is the identifier (name) of the Phrasecatalogue. It should be short and unique.
    Catalogue Version: The Catalogue Distributor is a text field which gives details about who has composed or distributed the Phrasecatalogue.
    Catalogue Date: This defines the data of issue for the current Phrsecatalogue.
    Number Range: This identifier is only needed for compatibility with SAP® systems. All mentioned fields are necessary for a Phrasecatalogue.

    Mapping of Catalogues

    One unique function of the pHrasetool by Rimpido is its capability to map (reference) to Phrasecatalogues against each other. I.e., the Phrases of the first Phrasecatalogue will be taken and for each Phrase a search will be executed trying to find a similar Phrase in the other Phrasecatalogue.
    These references can be used to match to Phrasecatalogues, to "copy" Phraseset-reference form one Phrasecatalogue to another or to add missing Phrases.

    Please Note: Prior to make use of the Mapping it is needed to understand the conditions of the mapping!

    Firstly, note that the Mapping process has a direction. Whenever youw ant to use a function on two mapped Phrasecatalogues, please be sure that you have maped the Phrasecdatalogues using the same direction. In that way if you map Phrasecatalogue A against Phrasecatalogue B, A is declared as "Origin Catalogue" and B is declared as "Target Catalogue"

    If you ever mix up the direction you'll get unexpected and unreliable results on the functions!

    The Mapping itself runs through the Phrases of the "Origin Catalogue" (A) and tries to find the same Phrase or a similar Phrase in the "Target Catalogue" B using the following search in hierarchical order:

    1. Same Phrasecode Search for any Phrase with the same Phrasecode
    2. Identical Phrase: Search for any identical Phrase
    3. Nearly Identical Phrase: Search for any identical Phrase (case-insensitive, ignoring ".", ",", ";", ":", and " ")
    4. Boolean TextSearch Search for any Phrase which is mostly similar using a fultlext search from MySQL with a minimum on the match Value.
    5. Natural TextSearch Search for any Phrase where most "words" are similar using a fulltext search from MySQL with a minimum on the match value.


    The search for a Phrase stops at the first search which gives at least one matching Phrase. Thus, the matching could be pretty fast of on both catalogues Phrases with Phrasecodes exist only.

    If one matching Phrase is existing the mapping will be written into the reference table, but only the first three search functions and hte fourth functionif a very hgh match criteria is met. For the forth and fifth search function the mapping will be normally be writteninot the temp table for a later review. Also, every matching with two or more matching Phrases will be written into the temp table. After a mapping process the temp table contains all the references which are ambiguous and therefore must be reviewed manually.

    You start a mapping process be clicking the butotn "Maping" on the index screen and thus selecting the "Origin Catalogue".

    On the first screen you can select the "Target Catalogue" as well a two other parameters. The two parameters may restrict hte mapping process. If the restriction cannot be fulfilled a second mathing will be made without the restriction.

    Restrict to the same Structure Code: This will restrict the matching process to a Structure Code. This structure code must be present in both Phrasecatalogues, e.g. two versions of hte EuPhraC where the structure Code is the EuPhraC Structure Code.

    Restrict to the Same Group: this will restrict the matching process to the Phrasegroup. This is useful if the Phrasecatalogues contain the same Phrases multiple times but in different Phrasegroups. If the matching catalogues uses Phrasegorups this option should be checked.
    The next screen is just an information screen the submitting will start the Mapping Process and that the following Process ma take some omre time. You may cancel the mapping only at this point as afterwards some references may be written.

    The automatic mapping process can be done in background. you may use any program and even your browser as long a this window is not effected. Internally, this functions uses AJAX- Requests to launch the mapping process. The current progress is indicated by the progressbar and the precetage value. It has two minutes neither progress nor value has changed, the mapping has run into a severe error. Please try again or ask your adminstrator for any help.

    After a successful run of the automatic mapping process, you'll get informed how many Phrases could be matched and how many Phrases must be reviewed manually.
    At this point (any at any later process) you may pause the review process and come back later.
    Submitting the button will ask you for the manual mapping for the first Phrase.
    At this prompt you may either select a reference, defer the Phrase for later review or pause the Mapping. You'll see the current Phrase to be mapped on the top of the Page. Following after are displayed all possible Phrases to select for referencing and above of them the option not to reference the current Phrase.
    By selecting one of the options and submitting the form the match will be written into the reference table.
    Sumbmitting the "Defer"-button will put the Phrase at the end of the list, so that it can be reviewed later.
    You can always pause the mapping, but use the button for it whenever possible. Each entry in the temp table will be locked to a user if it is displayed to a user.
    If you do not press the button, the lock will be removed for other users after two days.
    After pausing the mapping you'll get an information isplayed how to resume it. You can also select it from the main menu.
    Resuming the mapping will display a similar screen as the screen after the automatic process.
    You'll get an information how many Phrases have to be revived manually.

    Comparison of Catalogues

    Help is still under development!

    The Comparison is availabley for mapped Phrasecatalogues only! With this specialised search function it is possible to find mappings, differences in mappings and missing mappings between two Phrasecatalogues.
    On the initial screen you can select the options for your comparison. If you have already compared two catalgues during your session, the old values will be restored. First of all you have to select the "Origin Catalogue" and the "Target Catalogue" as well as a language. Only Phrases in the selected language will be displayed. If a reference is found but the Phrase is not found in the selected language, a link to the Phrasehead is displayed instead of the Phrase. You may restrict the comparison in idfferent ways. In that way you can control the sucess of the mapping process and make some minor corrections to it. Please note: The two Phrasecatlogues must be mapped against each other in the same way as selected in the search field. The direction of the mapping is critical for this as mentioned above.
    Assuming you interested in all references, you select "No restrictions". The selection form for the comparison will only be displayed if less than six entries were found, you may open it again by clicking the appropriate button. If there are any results they will be displayed in the list. This list prints out the Id, which is just an identifier, and the referenced Phrasetexts in the selected language of both catalogues. You may change the language in the search options to get more details about the reference. The list is separated in blocks of 25 records. The navigation bar is displayed below the listing. So you are able to review each and every mapping (reference) by scrolling and clicking through the pages. Additionally (if you have the fitting privileges) you may also view and delete the reference.
    Please note: It is not possible to recover any deletion process! Only if you really know what you are doing, than it is OK to use this funtions.
    A really useful restricted search option is the restriction to "Only in Origin Catalogue" In the result list only those Phrases are displayed where no mapped Phrase in the "Target Catalogue" exists. That gives the option to review the mapping process and looking for missing data. The main difference two the other screens is that on the "Target Catalogue" no data exists and thus only "NULL" is displayed. And instead of editing or deleting the reference you may copy the non-existing Phrase to a catalogue or add a reference using a custom search. By clicking on one of the buttons you'll be redirected to a new screen (see below). Also on this search it may be helpful to change the language to get further insights.
    Copying a Phrase does not only add an exact copy of a given Phrase to a catalogue but also add a refrence between the two Phrases. You may copy the Phrase from the "Origin Catalogue" to any catalogue you own. As default the "Target Catalogue" is selected. An exact duplicate will added to the selected catalogue and addtionally a referenc from the "Origin Catalogue" to the selected catalogue will be written, even if no mapping between bot catalogues has been done.
    Please note: This function does not check for any copyright issues. So you must be sure that copying a Phrase from the "Origin Catalogue" is not restricted by copyright laws (or any other law).
    Alternatively you may add a refernce using a custom search. Sometime there exists a similar Phrase in the "Target Catalogue" but the wording is quite different. In that circumstances the mapping process cannot find the possible reference. On the other hand you may use a special search on the Phrase finding a possible reference. It is recommended to use another tab for searching the Phrasetexts with the normal search function first and then copy the search string to this form. In that way it is most likely easier to find a possible reference. The search form presented in this dialogue is more restricted and less tuneable than the main search function. If any possible mapping was found, you'll get that displayed after submitting the search. Please select the best match and add a reference between "Origin Catalogue" and "Target Catalogue".
