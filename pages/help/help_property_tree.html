Help on Property Trees

This help document is still under development!
Further Information will be added in later versions of the software.
This section focuses on the concept of Propertytrees. Propertytrees and its inherit structures are a complex system to define a "Specification Database" where you may store all the data of (chemical) specifications. This data may include physical data, compositional information, chemical regulations, dangerous goods data, and further more. With the propertytrees you may setup different view on your data, in that way only part of the data is visible to a given user. There are serveral different conept to maintain this data, see below.
The data maintenance is done in a similar fashion for all subtopics, also theuser screens are very similar. Please take a look at the trees first, if you need an overview of the user interface itself.

    • General Info and Definition of Terms

    Propertytrees (as used in this software) are a concept to structure the visibilty of data from datasets as implemented in SAP R3 EHS-module.
    The concept of Propertytrees is complex and includes different aspects. These aspects are listed below:

    Propertytree (Tree) A Propertytree is a tree-like structure consisting of many Properties. On a system with the same Properties more than one Propertytree may exist. These trees defines the visible Properties and their hierarchical order when using the tree.
    Properties Properties define the settings for the nodes and leaves in a tree. There are several types of Properties but mostly only Node and Leaf is used. The Leaf always contains a Class and mostly they are labelled in the same way. Nodes and Leaves may not be mixed up.
    Classess The Classes are the link between Properties (and thus Propertytrees) and Attributes (which actually describe fields). Many Attributes are packed in one Class. The Class can be seen as their common topic/ heading, e.g. the Density.
    Attributes Attributes are always linked to a Class. The are referenced to one special field representing one characteristic aspect of the Class, e.g. the Unit of the Density. The Attribute is mostly labbeled as an extensio of the Class-labelling.
    Phrasesets Many Attributes are simply values but many other Attributes may only hold a selection of special Phrases, e.g. signal word for GHS-labelling. These selection of Phrases are called Phraseset. It is a set of Phrases which is valid for a Attribute. Mostly the Phraseset is labelled the same way a the Attribute.

    This software has the capability to use the different aspects from the table above. You may link the aspects against each other where appropriate.
    Please note: As this part does only make sense if you have a lot of data loaed in the software, it is a good start to upload your data via import function first. After you have uploaded one ore more import-files, you can manage your data in this software and profit from it's user interface.

    Trees

    A tree itself is just an identificator which can be labelled in different languages. The really useful data is in the reference between properties and trees which is the propertytree. The propertytree groups properties in a tree-like structure so that the properties form its nodes and leaves.
    First you have to add a new tree (giving it a name) to add properties afterwards. You can start using the index page, which lists all knwon trees. On that page you may view, edit, label or delete a tree and you also may view the propertytree of the selected tree.
    The labelling of elements is basically done always on the same type of form. On that form you'll get the main languages displayed in rows and you may click on the "Edit ..."-field which enables editing via JavaScript.
    If you have entered any text before, e.g. Rimpido Standard Selectiontree, this will be displayed. You may change this also by clicking on the text element. You must have JavaScript enabled to use this functionallity.

    Property Trees

    The Propertytree (as most other referencing tables) is displayed via a special tree-view. This tree view highly relies on JavaScript-technology, thus a modern, standard-compliant browser is higly recommended. On the initial view all nodes are collapsed. By clicking on the triangle of a node all of its leaves (which can also be nodes) will be displayed.
    You may rearrange the leaves and nodes via drag and drop, via the buttons of the menubar and via context menu (accessable via right mouse click). Please be patient while using this view. Some functions will need some time to get the data saved and displayed.
    Nodes are a special type of property (which defines its use as node).

    Please note: If you use a normal leaf as a node, you'll get informed in this view that you are using a property which ist not defined as node. You must either change the propety to be node or rearrange the tree. Below the Propertytree you'll find two listings of Properties.

    The first listing contains all Properties which are defined ads nodes, the Properties may contain other Properties below (in the tree structure). With these nodes you may arrange your Properties in the Tree hierarchically. You can easily add the Propetries via Drag'n'Drop. Dragging a Node will not remove it from the list as you may use the same node multiple times in one tree.
    The other listing contains all Properties which are not defined as leaves, thus these Propetries may not contain further Properties. Generally these Properties contain a reference to a class (see below) defining your attributes which shall be listed and maintained at this node. So you may now group your Property-leaves (and thus your classes) in different Property-nodes.
    By clicking the "tab close"-button, the current listing will be collapsed and hidden, so you can focus on one of the two listings. Furthermore you do not have to Drag'n'Drop the Properties over a long way with your mouse.
    There are some buttons / icons which will be explained below:

  • This is the search button, which submits a search on the current listing.
  • This is the delete button. After selecting a Phrase you can delete it from the Phraseset with this button or via rightclick.
  • This is the refresh button. You can refresh the whole listing with this button.
  • This is the root icon. The root icon is the base element in the lists, it cannot be deleted or dragged.
  • This is the folder icon. In this case the folder is equal to a node Property.
  • This is the property icon, it represents a Property (which is displayed in English).
  • This is the close button, which closes the listing giving you more space on the other listings.
  • This is the open button, which (re)opens the listing to make it visible.

    Properties

    Properties are the entries in a Propertytree. You can have more Properties than you have put in your Propertytrees and you can add a Property to more than one Propertytree.
    On the index page you'll get all your Propeerties displayed and additionally the corresponding class if it exists.
    Each property consists of many values, which must be set to get an properly configured Property:
    Property Code: This code is an expressie representation fo the Property. It may be used as an identifier in other systems (such as SAP®).
    Class: If your Type requires a Class, you have to select one here. Normally, the PropertyCode and the Class are the same.
    Type: You have to select a class from the list (list is shown below). The type defines the Property and puts it to the nodes or the leaves Allow Copies: Select this option if multiple copies of your Property may be used in your Specification Database. (Standard)
    Allow new entry: Select this option if new entries of your Property may be made in your Specification Database (Standard)
    Allow Changing: Select this option if your Property may be changes in your Specification Database (Standard)
    Allow Changing: Select this option if your Property may be changes in your Specification Database
    Individual Instance: Select this option if your Property may be used only in a single instance in your Specification Database.
    Internal: Select this option if your Property may be used internally only in your Specification Database.
    Profile: Add a Profile for your Property to be used in your Specification Database
    There are several Types which may be used for your Property:
    A: Standard Property. This is the standard leaf-property which must consist of a Class.
    B: This Property defines a listing. This is also a leaf-property which must consist of a Class.
    C: This Property defines a composition. This is also a leaf-property which must consist of a Class.
    D: This type is not in use at the moment.
    E: Node Property. This Property is a node and thus may not consist of a Class.
    F: Transport Classification. This Property is used as Classification of Transports.
    G: DG-additional data. This Property is used for additional DG-data for Dangerous Goods data.
    H: Hazard Induzing Substance. This Property defines the HIS for Dangerous Goods data.
    Normally you should select "A" or "E". If you select a different value, you have to be careful for any side-effect.
    Please note: At the moment the Type is not used be the Phrasetool by Rimpido, but there are used in other systems (e.g. SAP).
    As mentioned above the labelling of elements is basically done always on the same type of form. On that form you'll get the main languages displayed in rows and you may click on the "Edit ..."-field which enables editing via JavaScript. If you have entered any text before, e.g. Rimpido Standard Property, this will be displayed. You may change this also by clicking on the text element. You must have JavaScript enabled to use this functionallity.

    Classes

    Classes in this concept are more ore less only used to group Attributes. Thus the Propertytree with its Properties indirectly points to Attributes as all Attributes are grouped in Classes. The Classes used in this section must have a type of "100", which is also displayed on the index page.
    Adding or editing a Class is quit easy as you have to enter a Classe Code and the Type only. Normally the Class Code should be the same one (later) used in the Property. In this way you won't mix up the corresponding Properties and Classes and get confused be different Codes for corresponding elements. The Class Type should always be 100. This value does not effect the system (at the moment) but may effect Third Party systems.

    Attributes

    The information in your Specification Database may be of different type, e.g numeric values or characters, the attributes defines these parameters. Each Attribute must be grouped in a Class to be used. Attributes which are not referenced to a (used) Class cannot be access in your Specification Database at all. Additionally each Attribute can be labelled.
    As Attributes consist of some parameters, these must be set while adding or editing an Attribute. These parameters are as follows:
    Attribute Code: This is a expressive representation of the Attribute. Normally it is an extension to the Class Code staring with underscore (_).
    Type: Attribute may be of different Type: Characters, Numbers, Currency values, Dates and Times.
    Positions: The length of the value. In case of numbers and currency values it's only the pre-decimal places. Maximum is 30, Attributes of Type character which are longer than 29 characters must be set to 30.
    Decimal places: Only in use for numbers and currency values.
    Signed value: Must be set if you do not have positiv values only.
    Measurement Unit: If there is an internal maesurement unit in use fpr this Attribute, enter it here.
    Single Value: If the Attribute represents a single value, i.e. only value value may be set, this field must be checked.
    Intervall Value: If the Attribute represents an intervall, then check this field.
    Case-sensitive: If the Attribute represents a case-sensitive value, then check this field.
    Testing function: If the Attribute represents a value which has to be checked against a function, then add the function here.
    Testing table: Alternatively, you may check the value against a table, add this table here.
    Status: The Attribute may be of either "not released" (0) or "relesed" (1) status.
    As mentioned above the labelling of elements is basically done always on the same type of form. On that form you'll get the main languages displayed in rows and you may click on the "Edit ..."-field which enables editing via JavaScript.
    If you have entered any text before, e.g. Special Attribute, this will be displayed. You may change this also by clicking on the text element. You must have JavaScript enabled to use this functionality.

    Class-Attributes-References

    As you can manage the Properties to the Trees (Propertytrees) via Drag'n'Drop, you can also manage your Class-Attribute-References.
    Please note: These screens rely on JavaScript, a modern, standard-compliant browser is highly recommended.
    The screen is splitted in two views: On top you'll find all Classes (with the referenced Attributes) and below you'll find all Attributes. We start with the top screen, see right.
    In this listing you'll find all your Classes as nodes. Initially these nodes are collapsed, on clicking on the triangle the node opens and all referenced Attributes will be loaded on demand. You may rearrange and add Attributes via Drag'n'Drop or via context menu (accessable via right mouse click). You can also delete an Attribute from a Class by selecting it and clicking the delete button.
    The second screen gives you a list with all your attributes. You can add an Attribute to a Class via Drag'n'Drop in the upper listing and so you can fill all your Classes with the appropriate Attributes.
    Additionally (as in all listings) it is possible to search for Attributes using the search bar. You may ether search for an Attribute's labelling or for an Attribute's code. The search is expaned with wildcards (*) before and after the entered search string (the same applies to the search of Classes).

    Return to Top

    Phraseset-Attributes-References

    The Phraseset-Attribute-References are maintained the same way as the Class-Attribute-References, but additionally to the Phraseset also a Property must be added. You may use this screen the same way as all other tree-like structured screens: Drag'n'Drop for adding and rearranging elements, context menu or menu button for deleting and the search bar to restrict the displayed elements.
    You can add all the Phrasesets to Attributes first. But after your work is done, all Phrasesets must contain a Property! Take the two listings below the main listing to maintain Phrasesets and Properties in your Attributes.
    Please note: Normally one Attribute contains one Phraseset which contains one Property. This reference will be saved in a single row as an one-to-one-to-one relationship!

    Phrasesets

    As mentioned above the Phraseset is a Label for a special selection of Phrases. It is highly used for Attributes which are no values but may only hold a small selection of Phrases. Phrasesets are linked to Propertytrees as well as to Phrasecatalogues.
    A Phraseset which is linked to neither an Attribute nor a Phrase is not useful, because it acts as the linking element between the both aspects.
    Within the Phrasetool by Rimpido it is possible to manage the Phrasesets meaning to add, edit and remove Phrases to a Phraseset and to copy Phrasesets from one catalogue to another.

    Managing Phrasesets and Phrases

    If you click on "List Phrasesets" you are actually redirected to the Phraseset-Phrase-Reference index page. Normally you should have imported all necessary Phrasesets and Phraseset-Attribute-references via Propertytree-import.
    On that index page it is possible to add, edit, or remove Phrases from a Phraseset. In the upper half the Phraseset-Phrase-references are displayed. The Phrasesets are displayed as nodes, and clicking on the triangle opens the node (Phraseset) and displaying all the Phrases which are referenced to this Phraseset. You can restrict the displayed Phraseset by the search field. On the lower half all the Phrases restricted to 350 Phrase will be displayed. To add a Phrase just drag it with your mouse from the list and put it on one of the nodes (Phrasesets) in the upper list. A green mark should appear, then you can drop the Phrase. Aftwards the whole listing of Phrasesets will be refreshed. This may take a while. The Phrases can also be restricted by using the search field. Please note: The result will always be displayed in English but you may search in any language where you Phrase exists in. (This may be changed in later versions.)
    There are some buttons / icons which will be explained below:

  • Search Button This is the search button, which submits the search.
  • Delete Button This is the delete button. After selecting a Phrase you can delete it from the Phraseset with this button or via rightclick.
  • Refresh Button This is the refresh button. You can refresh the whole listing with this button.
  • Folder Icon This is the folder icon. In this case the folder is equal to a Phraseset.
  • Phrase Icon This is the Phrase icon, it represents a Phrase (which is displayed in English).
  • TabClose Button This is the close button, which closes the listing giving you more space on the other listings.
  • TabOpen Button This is the open button, which (re)opens the listing to make it visible.

    • Copy Phrasesets (From one catalogue to another)

    One unique function of the Phrasetool by Rimpido is the option to map two (or more) Phrasecatalogues against each other.
    With the Mapping functionality it is possible to use the Phrasesets from a Propertytree A, which is related to Phrasecatlogue A, with the Phrases from Phrasecatalogue B. Prior to use the Copy function you have to fully map the two Phrasecatalogues, i.e. automatic mapping and manual mapping must be done completely.
    Please note: The direction of the mapping is critical for the Copy process. If you take our example, you have to map Phrasecatalogue A (with the references to Propertytree A) as "Origin Catalogue" map it against Phrasecatalogue B as "Target Catalogue". Otherwise you copying will result in an error.
    You have to select the "Origin Catalogue" and the "Target Catalogue" and sumbitting the form. After submitting the form, the Copy function will start. It will go through all Phrasesets referenced to the "Origin Catalogue", looking for every Phrase using the Mapping, and will add a reference for that Phraseset to the mapped Phrase of the "Target Catalogue". If no Phrase of the "Target Catalogue" is mapped to the current Phrase of the "Origin Catalogue" no new reference will be written. This should result in many new references for the Phrasesets with the "Target Catalogue". If all Phrases of the "Origin Catalogue" could be mapped against a Phrase of the "Target Catalogue", the new references are equal in number to the old ones.
    The new references can be edited using the Phraseset Mangament Tool (see above) and afterwards they can be exported and used by other systems. During the copy process, a progress bar will be displayed.
    The progressbar shows how many Phrasesets (in percent) have been copied yet.
    As like the other backgroun processes, you have to wait until the progressbar comes to an end. You may use any other browser tab or programm as long as it does not effect this tab.
    Result screen after copying Phrasesets
    After a successful Copy process the results will be printed out. The result should normally reflect the Mapping process in that way that only mapped Phrases will result in a new Phraaseset-Phrase-reference.
    If you missed to match all Phrases some Phraseset may be empty and thus no new references can be created. In this example (seen right) only a tenth of all Phraseset could be copied as only about 15 percent of all Phrases were mapped.
    Please consider to use this function again if you have significantally added mappings between the two Phrasecatalogues.
