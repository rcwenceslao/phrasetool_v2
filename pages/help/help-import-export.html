HELP ON IMPORT AND EXPORT

The Phrasetool by Rimpido allows privileged user to import data into the system via well formatted files. The export is (in parts) available for more user groups.

The file formats used in import and export are basically the same. But not all formats are implemented on each import and export function. Please look at the import / export form to get detailed information which formats are used by this function.

The import as well as the export starts with a screen for selecting certain options (and uploading files in case of import). Then you will be sent to a screen with one or more progressbars. This page will trigger necessary function calls via AJAX-requests, so please do not close or reload the page unless after two minutes no progress can be detected.
The import and export may be very time consuming, so please be patient. On the other hand you may leave these functions unattended running in the background as lomg as you do not close the webbrowser or the webpage (or going in standby with your computer).

IMPORT

The import is devided in three parts: Phrasecatalogue and Phraseset. While the last two Import functions can import SAP exchange files only, the import of phrasecatalogues can make use of four different file formats. The files must(!) match the selected file format, otherwise the import may result in failure and the imported data is damaged and unstable!

Please note: Phrasesets (actually the reference between Phrasesets and Phrases) may only be uploaded after you have imported the Phrases via Phrasecatalogue import and the Phrasesets via Propertytree Import. The Phraseset file and the Phrasecatalogue file may be the same one. Firtst you must upload this file as a Phrasecatalogue and then you may add the references via importing as Phraseset file.
Instead of uploading a file via the file menu of your browser, it is also possible to upload a file in the upload directory directly. This may be needed if the import file is bigger than the maximum size limit given by the webserver. You should ask your administrator, if you want to use this option.



IMPORT OF PHRASECATALOGUES


There are several options and field which may be selected in order to get the desired import of a Phrasecatalogue.
File: If you have uploaded a file, select the right one. Otherwise chose a file from your local computer to upload and import it. There exists a limit of the file size and large files may be very slowly uploaded.
Type of Upload File: There is no automatic detection of the file type. You have to select the file type carefully as a wrong file type may result in irreproduceable results.
Separator: If you selected "Comma Separated Values" as filte-type, you have to enter the separator here. In your uploaded file the vallues must be separated (in one row) by this separator only (which must be the same in the whole file) and your values may not be enclosed by " or alike.
Phrasecatalogue: Prior to import a catalogue you have to create (add) a catalogue in which you want to import the phrases for all file types but SAP exchange format. As at the moment only SAP exchange format contains data of the catalogue itself.
Prefix to PhrasenId: In some systems you'll need to have PhraseIds according to a specific range of characters. You may add a prefix to all imported PhraseIds to match this range. (Not available on SAP import)
Commercial Catalogue: Selecting this button restricts the use of any phrases of the catalogue to your own company/employer. No phrase will ever be released to other catalogues. All imported phrases will be marked as "commercial", so this can always be identified as personal phrases.
Released Catalogue: Releasing a catalogue allows you to earn credit points for new phrases. The new phrases will be copied into the master catalogue and for each copied phrasetext you get a credit point. All phrases will be checked against known commerial phrasecatalogues. If you want to participate with your phrases in the phrase community, you have to select this checkbox.
Overwrite Existing, Phrases only: Normally nothing will be overwritten by an import. Selecting this checkbox will overwrite all phrasetetxt and phrasedheads (all language specific texts and the master data of a phrase) in the selected catalogue. But neither any existing catalogue data nor any existing phrasegroup data will be overwritten.
Overwrite Existing: As mentioned above, normally no data will be overwritten. If you select this checkbox ALL data in the selected catalogue will be overwritten with the data from the input file. This is only useful if you have already data in your catalogue.



IMPORT PROPERTYTREES


There are several options and field which may be selected in order to get the desired import of a Propertytree.
File: If you have uploaded a file, select the right one. Otherwise chose a file from your local computer to upload and import it. There exists a limit of the file size and large files may be very slowly uploaded.
Type of Upload File: There is no automatic detection of the file type. Currently the SAP exchange format is available only.
Tree: Prior to import a propertytree you may to create (add) a tree in which you want to import the data.
Overwrite Existing: As mentioned above, normally no data will be overwritten. If you select this checkbox ALL data in the selected propertytree will be overwritten with the data from the input file. This is only useful if you have already data in your propertytree.



IMPORT OF PHRASESETS


There are several options and field which may be selected in order to get the desired import of a Phraseset.
This phraseset import is only importing the references of phrasesets and phrases and neither the phrasesets nor the phrases. Often the import file for this import is identically with the import of the phrasecatalogue if you are using SAP exchange format. This is not an error but a design decision.

File: If you have uploaded a file, select the right one. Otherwise chose a file from your local computer to upload and import it. There exists a limit of the file size and large files may be very slowly uploaded.
Type of Upload File: There is no automatic detection of the file type. Currently the SAP exchange format is available only.
Phrasecatalogue: Prior to import the phraseset phrase references (via this phraseset import) the phrases and phrasesets must exist in the system. You have to select the desired phrasecatalogue to import the references.
The import will be restricted to phrases from the catalogue selected above and phrasesets belonging to your company. The phrasesets can be imported via the propertytree import, but they are not limited by one property tree.
Overwrite Existing: As mentioned above, normally no data will be overwritten. If you select this checkbox ALL data in the selected references will be overwritten with the data from the input file.



IMPORT PROCESSING


After submitting the main form,two progressbars will be displayed.
Overall progress: This progress bar gives you an idea how many steps will follow until the import is done. Some import filters will need less steps than others, so the progress bar may not reach the end on the right side.
Step progress: This progress shows detailed information about the current step. Larger import files will result in very many steps with little progress, so please take a look at the percent value to check if the import is still running.
Please note: The import process may be very time consuming! After at least 60 seconds should get a new information on the screen. Normally the percent value increases every 7 seconds, but some times this may be longer. Please be patient.
Please also note: You can put the browser window in background and may also use other browser tabs while the import is running. But the import browser window may not be affected from any disturbences or the import will break.
Broken imports: If your import failed, you may ask your administrator if he can help you by looking into the log files. Normally you can restart the import by chosing "Overwrite Existing". In that way your whole broken import will be overwritten. If that does not help, please ask your administrator.



FINISHED IMPORT


When your import is finished, a summary window is displayed. On that window a summary will be presented how many items where successfully added to the database and how many errors occured.
If you had selected an existing propertytree or phrasecatalogue a zero (0) will be displayed on both columns. Depending on the type of import and the format some values (for successes) may not represent the actual values as the only changes in the database count in.
Please note: There is no possibility to get detailed information about the errors within the webpages. Normally all imports will be recorded with log files, so please ask your administrator if you encounter any errors. If you forgot to select "Overwrite Existing" while overwriting an existing propertytree or phrasecatalogue will result in many errors.



EXPORT

The export is devided in four parts: Phrasecatalogue, Search Result, Phraseset and Propertytree. During the export a file will be created which can be downloaded afterwards. Search Results and Catalogues may be exported not only in SAP exchange format but also as an MS Excel document. Further export formats may be implemented at a later moment. The export files are compatible to the import filters.



EXPORT OF PHRASECATALOGUES


There are several options and field which may be selected in order to get the desired export of a Phrasecatalogue.
Phrasecatalogue: Select the phrasecatalogue you want to export and download.
Type of Download File: The export file will have the selected format. Phrasecatalogue information is stored in the SAP exchange format only.
Be aware, that the export of MS Excel files is both time and memeory consuming. You may reach the system's borders and that way the export may fail.
Target Operating System: Please select the target operating system. The file will have special line feed characters depending on the system. Binary formats (as MS Excel) do not neet this information.
All Languages: If selected (default), all phrases in all languages will be exportet. Otherwise you'll see an additional selectbox where you may select the desired languages. You may select multiple languages but all available languages will be displayed in the box at the moment.



EXPORT OF SEARCH RESULTS


This export may be useful if you want to export a small group of phrases only then use this function. First you have to search in the phrasentexts for phrases, afterwards you may export this result.
Type of Download File: The export file will have the selected format. Phrasecatalogue information is stored in the SAP exchange format only. Be aware, that the export of MS Excel files is both time and memeory consuming. You may reach the system's borders and that way the export may fail.
Target Operating System: Please select the target operating system. The file will have special line feed characters depending on the system. Binary formats (as MS Excel) do not neet this information.
Phrasetexts: You may add one or more phrasetexts by selecting them via checkbox. Below the submit button are two buttons which select all or none of the phrasetexts.
Please note: Due to some technical restrictions only 350 phrasetexts can be exported simultaneously. Thus the export form will only show the first 350 records of the search result.



EXPORT OF PROPERTYTREES


There are two options and field which may be selected in order to get the desired export of a Propertytree.
Tree: Select the propertytree you want to export and download. Please note: Currently only export in SAP exchange format is possible.
Target Operating System: Please select the target operating system. The file will have special line feed characters depending on the system. Binary formats (as MS Excel) do not neet this information.



EXPORT OF PHRASESETS


There are two options and field which may be selected in order to get the desired export of a phraseset phrase reference file.
Phrasecatalogue: Select the phrasecatalogue the phrases belong to which you want to export and download.
Please note: Currently only export in SAP exchange format is possible.
Target Operating System: Please select the target operating system. The file will have special line feed characters depending on the system. Binary formats (as MS Excel) do not neet this information.



EXPORT PROCESSING


After submitting the main form one progressbars will be displayed.
Overall progress: This progress bar gives you an idea how many steps will follow until the import is done. Some parts of the export may be faster and other may be slower but the export is decently faster than the import.
Please note: The export process may be time consuming also! After at least 60 seconds should get a new information on the screen. Normally the progressbar will be renewed every 7 seconds, but some times this may be longer. Please be patient.
Please also note: You can put the browser window in background and may also use other browser tabs while the import is running. But the import browser window may not be affected from any disturbences or the export will break.
Broken exports: If your export failed, try to export the same data again. If that does not help, please ask your administrator.



FINISHED EXPORT


During the export process in the background the webserver creates a files with the desired data. After a successfull export you will get a link to the export file. You can download this file at the same day multiple times, but do not forget to store the link as it will only displayed once after the export.



FILE FORMATS IN USE

The are several format defined for the use with the import and export fuinctions. Please note: Not every format is available on every funtion! Look at the page to learn which function is capable of which format, the availability of formats may be extended during the development of the software.
At least the SAP exchange format may be selected at every function.



SAP EXCHANGE FORMAT


One goal of the software is to be as much compliant to SAP EH&S-module as possible.
To achieve this, the SAP exchange format is implemented in all import and export funtions. This format is a text-based format and human readable (if you know much about the internal structure of SAP EH&S).
Each row contains is limited to 80 characters and a combination of "function code" a value. The function code defines the type of value. The rows are hierarchical ordered in that fashion that some "function codes" can be seen as headings. These headings may define the table the following data (in following rows) has to be stored into.
Further details can be acquired by SAP or in SAP-related literature.



MICROSOFT EXCEL IN COLUMNS


There are two different formats for your MS Excel documents, this description only include the one with languages in columns.
The import of MS Excel documents is restricted to well formated documents. You may use different versions of MS Excel documents (beginning with Excel 97 and including Excel 2013) as the system tries to detect the right version automatically. The first row must contain the headings and may contain any number of columns. If you do not include the headings in the first row, your import will fail. There are some columns which must appear in the document and some columns must contain data otherwise your import will also fail. The data is written in rows: One row will be imported as one Phrasehead with multiple Phrasetexts. Additionally a catalogue must be selected prior to importing an MS Excel document as there is no catalogue data defined within the document.
The following list describes the columns (column headings) neccessary and possible for your import document.

Phrasegroup The PhraseId, identifier of the Phrase. The heading is neccessary but you may leave the field blank to have a PhraseId generated. (PhrasenId)
PhraseId The Phrasegroup of the Phrase in the row. The heading is neccessary and the field in each row is neccessary. (Phrasengruppe)
Remark Remark to the Phrase. (Bemerkung)
Original Code Some Code of the Phrase in the native database, e.g. R65. (Original Code)
Phrasecode Code of the Phrase. Please note: The same Phrasecode will be applied to all languages. (Phrasencode)
Phrasestate Status of the Phrase (seldom used). (Phrasenstatus)
Graphics Filename (and path) of the graphicsfile. Please note: The same filename will be applied to all languages. (Grafik)
English The English translation of the Phrase. The heading is neccessary but you may leave this field empty. (Englisch)
Language The translation of the Phrase in the language, e.g. German, Norwegian, Hungarian. The language will be identified through the database and the name in the heading must be the same as in the database. (Deutsch, Norwegisch, Ungarisch)

The headings must have the same case-sensitivity as written in the list above. Any differences may result in failures during import.
You may use either English or German headings but no other language as they cannot be detected. German headings are given in brackets. The language must be set in the database, so you may use any language name as long as it is set in the database. The matching of languages is case-insensitive.
If you leave a field in a language empty no data will be written for this row in that language. This may be useful if you have not a complete translation of everything.
But be aware that the import will not fail if you do not have any language field of a given row filled with data. This will result in an orphaned phrase with no phrasetexts.

EUPHRAC EXPORT


The EuPhrac is the European Standard Phrases Catalogue for (extended) Safety Data Sheets and Exposition Scenarios. It can been downloaded from www.esdscom.eu. All rights on the Phrases, the document and its structure remain to the owners of the EuPhraC, see also the General Terms. You can upload the EuPhraC document directly into the Phrasetool by Rimpido without reformating the file (up to version 1.2).
Only a catalogue must be selected prior to importing an EuPhraC file as there is noch catalogue data defined within the document.
The format of the EuPhraC is basically a special type of CSV. Instead of using commas, tabs are used as separator. You may import other files with this import filter as well, but it is only compliant to the EuPhraC.
